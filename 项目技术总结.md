# ELEVEN Blog Tuner - AI 应用开发技术总结

## 项目概述

ELEVEN Blog Tuner 是一个基于多 Agent 架构的智能博客写作系统，采用 Python + FastAPI 后端和 Vue3 前端技术栈。系统通过 RAG（检索增强生成）技术构建个人知识库，利用多 Agent 协作完成风格学习、文章生成和质量审查等任务。

---

## 1. MCP (Multi-Component Pipeline) 架构与实现

### 1.1 架构设计理念

MCP（多组件管道）架构是本项目的核心设计模式，通过将复杂的 AI 工作流程拆分为独立的、可复用的组件，实现模块化的系统架构。

### 1.2 核心组件设计

```python
class MCPTools:
    """MCP 工具管理器"""
    
    def __init__(self):
        self._tools: Dict[str, Any] = {}
    
    def register_mcp_tool(self, name: str, tool: Any):
        """注册 MCP 工具"""
        self._tools[name] = tool
    
    def get_mcp_tool(self, name: str) -> Any:
        """获取 MCP 工具"""
        return self._tools.get(name)
```

### 1.3 管道编排机制

系统采用分层管道架构：

```
输入层 → 处理层 → 分析层 → 生成层 → 输出层
   ↓        ↓        ↓        ↓        ↓
文档清洗  语义分块  向量检索  内容生成  质量审查
```

**关键技术点**：
- **组件解耦**：每个组件通过标准接口通信，降低耦合度
- **数据流管理**：使用元数据传递上下文信息，保持数据一致性
- **错误处理**：每个组件独立处理异常，防止单点故障扩散

## 2. Function Call 机制设计与集成

### 2.1 工具调用架构

基于 BaseAgent 实现统一的工具调用接口：

```python
class BaseAgent:
    def __init__(self, ...):
        self.tools: Dict[str, Callable] = {}
    
    def add_tool(self, name: str, func: Callable) -> "BaseAgent":
        """注册工具"""
        self.tools[name] = func
        return self
    
    async def call_tool(self, name: str, **kwargs) -> Any:
        """调用工具（带缓存机制）"""
        # 生成缓存键
        cache_key = f"{name}:{json.dumps(kwargs, sort_keys=True)}"
        cache_key_hash = hashlib.md5(cache_key.encode()).hexdigest()
        
        # 检查缓存
        cached_result = tool_cache.get(cache_key_hash)
        if cached_result is not None:
            return cached_result
        
        # 执行工具
        result = await self._execute_tool(tool_func, **kwargs)
        
        # 缓存结果
        tool_cache.set(cache_key_hash, result)
        return result
```

### 2.2 工具注册模式

每个 Agent 在初始化时注册专属工具：


### 2.3 异步执行与线程池

```python
async def _execute_tool(self, tool_func: Callable, **kwargs) -> Any:
    """在线程池中执行同步工具函数"""
    if asyncio.iscoroutinefunction(tool_func):
        return await tool_func(**kwargs)
    else:
        # 使用线程池执行同步函数
        loop = asyncio.get_event_loop()
        return await loop.run_in_executor(
            thread_pool.executor, 
            lambda: tool_func(**kwargs)
        )
```

**技术优势**：
- **缓存机制**：避免重复计算，提升响应速度
- **异步支持**：支持异步和同步工具函数
- **线程池管理**：防止阻塞主事件循环

```python
class AgentProtocol:
    def __init__(self, enable_circuit_breaker: bool = True):
        self.call_chain = AgentCallChain()
        self.circuit_breaker = CircuitBreaker() if enable_circuit_breaker else None
    
    async def execute_task(
        self,
```

### 3.3 熔断机制实现

```python
class CircuitBreaker:
    """熔断器 - 防止级联故障"""
    
    def __init__(self, failure_threshold: int = 5, recovery_timeout: int = 30):
        self.failure_threshold = failure_threshold  # 失败阈值
        self.recovery_timeout = recovery_timeout    # 恢复超时（秒）
        self._status: Dict[str, CircuitBreakerStatus] = {}
        self._failure_counts: Dict[str, int] = {}
    
    def can_execute(self, agent_name: str) -> bool:
        """检查是否可以执行"""
        status = self.get_status(agent_name)
        if status == CircuitBreakerStatus.OPEN:
            return False
        if status == CircuitBreakerStatus.HALF_OPEN:
            return self._half_open_calls[agent_name] < self.half_open_max_calls
        return True
    
    def record_failure(self, agent_name: str):
        """记录失败"""
        self._failure_counts[agent_name] += 1
        if self._failure_counts[agent_name] >= self.failure_threshold:
            self._status[agent_name] = CircuitBreakerStatus.OPEN
```

**状态流转**：
```
CLOSED (正常) → 失败5次 → OPEN (熔断, 30秒)
                              ↓
                    超时后 → HALF_OPEN (试探)
                              ↓
                    成功3次 → CLOSED (恢复)
                    失败 → OPEN (重新熔断)
```

## 4. RAG 知识库构建方法论

### 4.1 数据处理管道

RAGPipeline 实现完整的文档处理流程：

```python
class RAGPipeline:
    def __init__(self):
        self.washer = DocumentWasher()      # 文档清洗
        self.chunker = Chunker(...)         # 语义分块
        self.embedder = EmbeddingService()  # 向量化
        self.searcher = Searcher()          # 向量检索
        self.reranker = Reranker()          # 结果重排
    
    async def process_document(self, doc: str, metadata: Dict = None) -> bool:
        # 1. 文档清洗
        clean_doc = self.washer.wash(doc, file_type=file_type)
        
        # 2. 智能分块（带元数据）
        chunks_with_metadata = self.chunker.split(
            clean_doc, 
            strategy="semantic", 
            metadata=metadata
        )
        
        # 3. 批量向量化
        embeddings = await self.embedder.embed_batch(chunks)
        
        # 4. 索引存储
        await self.searcher.index(chunks, embeddings, metadatas)
```

## 5. RAG 知识检索策略与优化

### 5.1 两阶段检索架构

```python
async def search(self, query: str, top_k: int = 5) -> List[SearchResult]:
    # 阶段1：向量检索（粗排）
    query_emb = await self.embedder.embed(query)
    candidates = await self.searcher.search(query_emb, top_k=top_k*2)
    
    # 阶段2：重排序（精排）
    results = await self.reranker.rerank(query, candidates, top_k=top_k)
    return results
```

### 5.2 向量检索实现

```python
class Searcher:
```

### 5.3 重排序优化

支持 Cross-encoder 精排：

```python
class Reranker:
    def __init__(self):
        self.use_local = settings.use_local_rerank
        self.model = "bbjson/bge-reranker-base"
    
    async def rerank(
        self, 
        query: str, 
        candidates: List[SearchResult], 
        top_k: int = 5
    ) -> List[SearchResult]:
        if self.use_local:
            # 使用本地 Cross-encoder
            scores = self._cross_encoder_score(query, candidates)
        else:
            # 使用简单排序降级
            scores = self._simple_score(query, candidates)
        
        # 按分数排序并返回 Top-K
        return sorted(candidates, key=lambda x: x.score, reverse=True)[:top_k]
```

## 6. 记忆系统实现

### 6.1 短期记忆（对话上下文）

```python
class Memory:
    """对话记忆 - 基于双端队列"""
    
    def __init__(self, max_history: int = 10, max_tokens: int = 4096):
        self.max_history = max_history
        self.max_tokens = max_tokens
        self.history: deque = deque(maxlen=max_history)
    
    def add(self, role: str, content: str, **kwargs):
        """添加对话记录"""
        message = {"role": role, "content": content}
        message.update(kwargs)
        self.history.append(message)
        self._manage_context()  # 自动管理长度
    
    def _manage_context(self):
        """Token 长度管理"""
        total_tokens = 0
        temp_history = []
        
        # 从最新消息开始，保留不超过 token 限制的消息
        for message in reversed(self.history):
            content = message.get("content", "")
            message_tokens = len(content) * 0.25  # 粗略估算
            
            if total_tokens + message_tokens > self.max_tokens:
                break
            
            temp_history.append(message)
            total_tokens += message_tokens
        
        self.history = deque(reversed(temp_history), maxlen=self.max_history)
```

### 6.2 长期记忆（向量数据库）

```python
class LongTermMemory:
    """长期记忆 - 基于 ChromaDB"""
    
    def __init__(self, vector_db_path: str, embedding_model: str):
        self.client = chromadb.PersistentClient(path=vector_db_path)
        self.collection = self.client.get_or_create_collection(
            name="long_term_memory",
            metadata={"hnsw:space": "cosine"}
        )
    
    async def store(self, key: str, content: str, metadata: dict = None):
        """存储记忆"""
        embedding = await self._get_embedding(content)
        self.collection.add(
            ids=[key],
            documents=[content],
            embeddings=[embedding],
            metadatas=[metadata]
        )
    
    async def retrieve(self, query: str, top_k: int = 5) -> List[str]:
        """检索相关记忆"""
        query_embedding = await self._get_embedding(query)
        results = self.collection.query(
            query_embeddings=[query_embedding],
            n_results=top_k
        )
        return results.get("documents", [[]])[0]
```

```python
class LLMFactory:
    """LLM 工厂 - 支持多提供商"""
    
    _providers = {
        "openai": OpenAIProvider,
        "local": OllamaProvider
    }
    
    @classmethod
    def create(cls, provider: str = None, **kwargs) -> BaseLLM:
        if provider is None:
            provider = get_settings().llm_provider
        
        provider_class = cls._providers.get(provider)
        if not provider_class:
            raise ValueError(f"不支持的提供商: {provider}")
        
        return provider_class(**kwargs)
```

## 9. 开发经验总结

### 9.1 大模型应用开发经验

1. **Prompt 工程**：系统提示词设计遵循"角色定义 + 职责说明 + 工作流程"模式
2. **容错设计**：LLM 输出不稳定，需要多层验证和降级策略
3. **成本控制**：通过缓存和批量处理降低 API 调用成本

### 9.2 RAG 系统优化经验

1. **分块策略**：语义分块优于固定长度分块，保持上下文连贯性
2. **检索优化**：两阶段检索（向量检索 + 重排序）显著提升准确性
3. **元数据利用**：充分利用元数据过滤，提高检索相关性

### 9.3 Agent 系统设计经验

1. **职责分离**：每个 Agent 专注单一职责，通过协作完成复杂任务
2. **状态管理**：熔断机制防止级联故障，提升系统稳定性
3. **可观测性**：详细的日志和监控，便于问题定位
